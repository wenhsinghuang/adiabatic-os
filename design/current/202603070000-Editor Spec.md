# 202603070000 Editor Spec — Obsidian-style MDX WYSIWYG

MDX text 是唯一 source of truth，no hidden state（DD11）。

---

## 兩種模式

| Mode | 用途 | 工具 |
|------|------|------|
| **View** | 預設。WYSIWYG rendering + inline editing | MDX compile → React render → contentEditable |
| **Source** | Power user / AI / 複雜編輯 | Monaco，raw MDX |

---

## 編輯模型：三條規則

### Rule 1: Text → Obsidian-style

任何渲染成文字的東西。點進去 → 顯示 source → 直接編輯 → blur → re-render。

涵蓋：
- Markdown formatting（`**bold**`、`_italic_`、`` `code` ``）
- Expression（`{1+4}` 顯示 `5`，點進去顯示 `{1+4}`）
- Link（`[text](url)` 顯示 rendered link，點進去顯示 markdown source）
- Mixed JSX/MD 裡的文字部分

跟隨連結 → Cmd+click。

### Rule 2: Component → name bar

Standalone live React app。點擊 = 跟 app 互動，不改變行為。

Name bar（整條可抓 = DnD drag handle）：
- 顯示 component name
- Edit button → popover textarea 編輯 raw JSX
- Delete button
- Resize → 拖邊角改大小

Resize serialize 成 native JSX：`<div style={{width: "500px", height: "300px"}}><Component /></div>`

### Rule 3: Mixed JSX/MD → 整塊變 source

點進 mixed block 的文字部分 → 整個 block（包含 JSX tags）都顯示 source → 直接編輯 → blur → re-render。

```
View:    ┌─ Callout ────────────┐
         │ some bold text       │
         └──────────────────────┘

Click:   <Callout>some **bold** text</Callout>

Blur:    ┌─ Callout ────────────┐
         │ some bold text       │
         └──────────────────────┘
```

Mixed block 裡的 component 部分（rendered app）：點擊 = 跟 app 互動，不進入 edit mode。只有點文字部分才會觸發整塊變 source。

---

## Component 操作

| 操作 | 行為 |
|------|------|
| **顯示** | Rendered app + name bar（hover 顯示） |
| **DnD** | 抓 name bar 任意位置拖拽排序 |
| **Edit** | Name bar Edit button → popover textarea → 修改 raw JSX → save → recompile |
| **Delete** | Name bar Delete button → 移除 JSX from source |
| **Resize** | 拖邊角 → 更新 width/height → serialize 成 `<div style={{...}}>` wrapper |
| **Insert** | Slash command → 選擇 component → 插入 JSX + import |

---

## 隱藏的內容

| 內容 | View mode | Source mode |
|------|-----------|-------------|
| Import/export 行 | 隱藏，round-trip 保留 | 可見可編輯 |

Import/export 沒有 rendered output，自然無法在 view mode 點擊編輯。不是 special case，是 Rule 1 的自然結果。

---

## MDX Evaluation

- MDX 原生支援的都該支援
- `system` object 注入 MDX runtime scope
- Expression `{await system.query("SELECT ...")}` 直接 evaluate
- Async 用 MDX 原生能力處理

---

## Pipeline

```
MDX text (source of truth, stored in DB)
  ↓ compile (client-side, @mdx-js/mdx)
React component tree
  ↓ render with component mapping
Rendered page (WYSIWYG)
  ↓ user edits (contentEditable / popover / DnD / resize)
Patch MDX text
  ↓ auto-save (debounced)
DB (source of truth)
```

---

## 使用頻率（影響 UX 優先級）

1. 看（最常）
2. Markdown 編輯（常）
3. JSX 編輯（偶爾，通常是 AI 做）

---

## 關鍵 Invariant

1. **Round-trip 無損** — MDX → render → edit → serialize 回 MDX，不丟內容、不加 escape、不破壞結構
2. **No hidden state** — 所有可見狀態都能在 MDX text 裡找到對應（DD11）
3. **Auto-save** — debounced patch → save to DB
4. **Component isolation** — component crash → error boundary，不影響其他
